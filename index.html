Les fonctions d’agrégation et les instructions SQL associées

Les fonctions d’agrégation permettent d’effectuer un calcul sur plusieurs lignes d’une table et de retourner une seule valeur comme résultat. Elles sont utilisées pour analyser et résumer les données.

Fonctions d’agrégation

  • SUM() : calculer la somme
  • AVG() : calculer la moyenne
  • MIN() : trouver la valeur minimale
  • MAX() : trouver la valeur maximale
  • COUNT() : compter les enregistrements

Instructions SQL associées

  • GROUP BY : regrouper les lignes selon une colonne
  • HAVING : filtrer les groupes créés par GROUP BY

Ce que nous avons appris

  • Les fonctions d’agrégation permettent de résumer les données.
  • Elles retournent une seule valeur à partir de plusieurs lignes.
  • Les instructions GROUP BY et HAVING permettent d’organiser et filtrer les résultats.

La fonction MIN()

MIN() est une fonction d’agrégation en SQL qui permet de retourner la plus petite valeur d’une colonne dans une table.

À retenir

  • MIN() est une fonction d’agrégation.
  • Elle retourne la plus petite valeur d’une colonne.
  • Elle peut être utilisée avec des nombres, des dates ou du texte.
  • Les valeurs NULL ne sont pas prises en compte.

Cas d’utilisation

  • Trouver le salaire minimum des employés.
  • Afficher la date la plus ancienne d’un projet.
  • Trouver la première valeur alphabétique dans une colonne texte.

Syntaxe

SELECT MIN(nom_colonne)
FROM nom_table;

Exemple simple

Afficher le plus petit salaire des développeurs. 


SELECT MIN(Salaire)
FROM developpeur;
NomDev Salaire
Ali 5000
Sara 7000
Amine 8000

Résultat : MIN(Salaire) = 5000

La fonction MAX()

MAX() est une fonction d’agrégation en SQL qui permet de retourner la plus grande valeur d’une colonne dans une table.

À retenir

  • MAX() est une fonction d’agrégation.
  • Elle retourne la plus grande valeur d’une colonne.
  • Elle peut être utilisée avec des nombres, des dates ou du texte.
  • Les valeurs NULL ne sont pas prises en compte.

Cas d’utilisation

  • Trouver le salaire maximum des employés.
  • Afficher la date la plus récente d’un projet.
  • Trouver la dernière valeur selon l’ordre alphabétique.

Syntaxe

SELECT MAX(nom_colonne)
FROM nom_table;

Exemple simple

Afficher le salaire le plus élevé des développeurs. 


SELECT MAX(Salaire)
FROM developpeur;
NomDev Salaire
Ali 5000
Sara 7000
Amine 8000

Résultat : MAX(Salaire) = 8000

La fonction COUNT()

COUNT() permet de compter le nombre d’enregistrements dans une table. Elle possède plusieurs variantes selon le type de comptage.

À retenir

  • COUNT(*) : compte toutes les lignes.
  • COUNT(colonne) : compte les valeurs non NULL.
  • COUNT(DISTINCT colonne) : compte les valeurs différentes.
  • Les valeurs NULL ne sont pas comptées avec COUNT(colonne).

Cas d’utilisation

  • Compter le nombre d’employés dans une table.
  • Compter les valeurs uniques d’une colonne.
  • Analyser des données avec doublons.

Syntaxe


SELECT COUNT(*) FROM table;

SELECT COUNT(colonne) FROM table;

SELECT COUNT(DISTINCT colonne) FROM table;

Exemple

Exemple avec valeurs répétées et NULL. 


SELECT COUNT(*) FROM developpeur;
SELECT COUNT(NomDev) FROM developpeur;
SELECT COUNT(DISTINCT NomDev) FROM developpeur;
NumDev NomDev
1 Ali
2 Sara
3 Sara
4 NULL

COUNT(*) = 4

COUNT(NomDev) = 3

COUNT(DISTINCT NomDev) = 2

La fonction SUM()

SUM() calcule la somme des valeurs d’une colonne. SUM(DISTINCT) calcule la somme des valeurs différentes en ignorant les doublons.

À retenir

  • SUM() calcule la somme de toutes les valeurs.
  • SUM(DISTINCT) calcule la somme des valeurs uniques.
  • SUM(DISTINCT) ignore les doublons.
  • Les valeurs NULL ne sont pas prises en compte.

Cas d’utilisation

  • Calculer le total des salaires.
  • Éviter de compter les valeurs répétées.
  • Analyser des données contenant des doublons.

Syntaxe


SELECT SUM(colonne)
FROM table;

SELECT SUM(DISTINCT colonne)
FROM table;

Exemple

Exemple avec des valeurs répétées. 


SELECT SUM(Salaire) FROM developpeur;

SELECT SUM(DISTINCT Salaire) FROM developpeur;
NomDev Salaire
Ali 5000
Sara 5000
Amine 8000

Résultat SUM : 5000 + 5000 + 8000 = 18000

Résultat SUM(DISTINCT) : 5000 + 8000 = 13000

La fonction AVG()

AVG() est une fonction d’agrégation en SQL qui permet de calculer la moyenne des valeurs d’une colonne numérique dans une table.

À retenir

  • AVG() est une fonction d’agrégation.
  • Elle calcule la moyenne des valeurs d’une colonne.
  • Elle fonctionne avec des données numériques.
  • Les valeurs NULL ne sont pas prises en compte.

Cas d’utilisation

  • Calculer le salaire moyen des employés.
  • Analyser la moyenne des ventes d’un produit.
  • Calculer la moyenne des notes d’étudiants.

Syntaxe

SELECT AVG(nom_colonne)
FROM nom_table;

Exemple simple

Calculer le salaire moyen des développeurs. 


SELECT AVG(Salaire)
FROM developpeur;
NomDev Salaire
Ali 5000
Sara 7000
Amine 8000

Résultat : AVG(Salaire) = 6666.67

La fonction GROUP_CONCAT()

GROUP_CONCAT() est une fonction d’agrégation en SQL qui permet de regrouper plusieurs valeurs d’une colonne en une seule chaîne de texte.

À retenir

  • GROUP_CONCAT() est une fonction d’agrégation.
  • Elle combine plusieurs valeurs dans une seule chaîne.
  • Les valeurs sont séparées par une virgule par défaut.
  • Elle est souvent utilisée avec GROUP BY.

Cas d’utilisation

  • Afficher plusieurs noms dans une seule cellule.
  • Regrouper les développeurs d’un même projet.
  • Afficher les produits d’une commande.

Syntaxe

SELECT GROUP_CONCAT(nom_colonne)
FROM nom_table;

Exemple simple

Afficher tous les noms des développeurs dans une seule ligne. 


SELECT GROUP_CONCAT(NomDev)
FROM developpeur;
--Exemple avec SEPARATOR
SELECT GROUP_CONCAT(NomDev SEPARATOR ' | ')
FROM developpeur;
NomDev
Ali
Sara
Amine

Résultat : Ali, Sara, Amine // Résultat : Ali | Sara | Amine

GROUP BY

GROUP BY est une instruction SQL qui permet de regrouper les lignes d’une table selon une ou plusieurs colonnes.

À retenir

  • GROUP BY regroupe les lignes ayant la même valeur.
  • Chaque valeur crée un groupe.
  • Elle est souvent utilisée avec les fonctions d’agrégation.
  • Elle permet d’analyser les données par catégorie.

Fonctions utilisées

  • COUNT()
  • SUM()
  • AVG()
  • MAX()
  • MIN()

Syntaxe

SELECT colonne, fonction(colonne)
FROM table
GROUP BY colonne;

Exemple simple

Nombre de développeurs par ville. 


SELECT ville, COUNT(*)
FROM developpeur
GROUP BY ville;
Ville NomDev
Fes Ali
Fes Sara
Casa Amine
Casa Omar
Rabat Yassine

Résultat : Fes → 2 | Casa → 2 | Rabat → 1

HAVING

HAVING est une clause SQL utilisée pour filtrer les groupes après l’utilisation de GROUP BY.

À retenir

  • HAVING filtre les groupes créés par GROUP BY.
  • Elle est utilisée avec les fonctions d’agrégation.
  • Elle s’exécute après GROUP BY.
  • Elle permet de garder seulement les groupes qui respectent une condition.

Cas d’utilisation

  • Afficher les villes ayant plus de développeurs.
  • Filtrer les groupes selon une moyenne ou une somme.
  • Analyser les résultats d’un GROUP BY.

Syntaxe

SELECT colonne, fonction(colonne)
FROM table
GROUP BY colonne
HAVING condition;

Exemple simple

Afficher les villes ayant plus de 1 développeur. 


SELECT ville, COUNT(*)
FROM developpeur
GROUP BY ville
HAVING COUNT(*) > 1;
Ville NomDev
Fes Ali
Fes Sara
Casa Amine
Casa Omar
Rabat Yassine

Résultat : Fes → 2 | Casa → 2

Ordre d’exécution et rôle de HAVING

Comprendre l’ordre d’exécution SQL permet de savoir pourquoi HAVING est utilisé après GROUP BY.

Ordre d’exécution

  1. FROM
  2. WHERE
  3. GROUP BY
  4. HAVING
  5. SELECT
  6. ORDER BY

Remarque

  • WHERE s’exécute avant GROUP BY
  • HAVING s’exécute après GROUP BY

Pourquoi pas WHERE ?

❌ Requête incorrecte

SELECT ville, COUNT(*)
FROM developpeur
WHERE COUNT(*) > 1
GROUP BY ville;

✅ Requête correcte

SELECT ville, COUNT(*)
FROM developpeur
GROUP BY ville
HAVING COUNT(*) > 1;
Explication :
WHERE ne peut pas utiliser COUNT() car COUNT() est calculé après GROUP BY.
